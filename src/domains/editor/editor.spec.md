# editor 行为规格

## 概述

editor 域当前负责 Tab Completion 能力：在编辑器中按触发键请求 AI 续写，并把建议以 Ghost Text 形式展示给用户。

## 核心行为

### 行为 1：按触发键请求续写

- **触发条件**：用户在可编辑的编辑器中按下配置好的触发键。
- **预期结果**：域服务构建上下文、选择 provider、请求 AI 建议，并把加载态与建议态同步到 CodeMirror 状态字段。
- **边界情况**：
  - 当编辑器是只读模式时 → 显示 Notice 提示，不发起请求。
  - 当未配置 provider 时 → 显示 Notice 提示，不发起请求。
  - 当重复快速触发时 → 应用防抖，忽略 300ms 内重复请求。

### 行为 2：确认或取消建议

- **触发条件**：用户在建议显示期间按 Enter、再次按触发键，或在锚点位置继续编辑。
- **预期结果**：Enter 或二次触发键会插入建议文本；Escape 或锚点编辑会取消建议并清空状态。
- **边界情况**：
  - 当建议为空字符串时 → 清除加载态，不插入任何内容。
  - 当正在加载时再次触发 → 不重复发起新请求。

### 行为 3：保持上下文格式

- **触发条件**：AI 返回原始续写文本后进入后处理。
- **预期结果**：列表、引用、表格和代码块上下文应尽量保留格式，必要时自动补前导换行、列表前缀和引用前缀。
- **边界情况**：
  - 当 AI 返回 Markdown 代码块包裹时 → 去掉外层 ``` 包裹。
  - 当配置为连续使用模式时 → 放宽句子长度限制。

## 不做什么（显式排除）

- 该域**不负责**配置持久化，设置读写由 settings / 命令层适配处理。
- 该域**不直接访问** Obsidian API，所有 Notice 与系统提示词读取都通过 Provider 注入。
- 该域**不负责** provider 选项组装，legacy provider 桥接留在命令层。

## 依赖

- 依赖的 Provider：ObsidianApiProvider、EventBus
- 依赖的其他域：无

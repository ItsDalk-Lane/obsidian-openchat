## 修复记录
### 1. Provider 默认行为回归修复
- **问题**：多个 Provider 与设置面板把 `enableReasoning` / `enableThinking` 从默认关闭改成默认开启，破坏向下兼容。
- **规则来源**：全局 CLAUDE.md 向下兼容原则；项目 CLAUDE.md 默认保持稳定行为要求。
- **修复方式**：恢复各 Provider 默认配置与设置 UI 中的空值回退为 `false`，避免旧配置或新建配置在未显式开启时改变行为。
- **影响文件**：src/LLMProviders/{azure,claude,deepSeek,grok,kimi,ollama,openRouter,poe,qianFan,qwen,siliconflow,zhipu}.ts；src/components/settings-components/provider-config/*。

### 2. Provider 模型加载错误提示治理
- **问题**：Provider 模型选择流程存在直接抛英文错误、原样显示 `error.message` 的情况，且缺少友好提示。
- **规则来源**：项目 CLAUDE.md 第 10 条错误处理；第 13.1 条双语支持。
- **修复方式**：将空模型列表改为返回空集合并统一走 i18n Notice；401/403 保持专门提示，其他错误统一落到友好提示。
- **影响文件**：src/components/settings-components/provider-config/ProviderGroupConfigModalController.ts；src/i18n/ai-runtime/locale/*。

### 3. OpenRouter / Poe / 文件系统工具链类型与文案修复
- **问题**：增量改动引入了 OpenRouter SDK 参数类型不匹配、Poe/Filesystem 类型收窄错误，以及多处用户可见硬编码文案。
- **规则来源**：项目 CLAUDE.md TypeScript 规范、双语支持要求；全局 CLAUDE.md 异常处理标准。
- **修复方式**：
  - OpenRouter：剥离 SDK 不接受的内部字段，修复请求类型；
  - Poe：补齐工具循环/请求读取/i18n 错误提示与类型收窄；
  - Filesystem/MCP Tool Hints：工具提示、warning、参数校验消息全部接入 i18n。
- **影响文件**：src/LLMProviders/openRouterRequest.ts、src/LLMProviders/openRouterResponses.ts、src/LLMProviders/poe*.ts、src/tools/vault/*、src/services/mcp/toolHints.ts、src/i18n/ai-runtime/*。

### 4. 用户侧错误暴露收口
- **问题**：Quick Actions、MCP 连接、聊天生成等用户流程仍存在直接把底层错误消息透传给 Notice 的情况。
- **规则来源**：项目 CLAUDE.md 第 10 条错误处理；全局 CLAUDE.md 异常处理标准。
- **修复方式**：改为 `DebugLogger` 记录详细错误，Notice 仅显示可翻译的友好提示；聊天生成失败使用统一错误文案，不再把 provider 底层消息直接呈现给用户。
- **影响文件**：src/components/settings-components/quick-actions/listRenderer.ts、src/services/mcp/McpClient.ts、src/core/chat/services/{ChatSessionManager,chatGeneration}.ts。

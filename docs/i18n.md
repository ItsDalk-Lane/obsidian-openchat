# 双语支持规范（中文 / English）

> 插件必须同时提供中文和英文两个语言版本。默认语言为中文。

## 强制规则

1. **所有面向用户的内容**必须提供中英文双语版本，包括但不限于：
   - 设置项标题与说明
   - 命令名称
   - 按钮文案
   - 提示信息（Notice）
   - 错误提示
   - Modal / View 文案
   - 占位符文本、引导文案、帮助说明

2. **新增功能时**，禁止只实现单语文案。所有新增用户可见文本必须
   同步提供中英文版本。

3. **默认展示中文**。当后续接入语言切换或语言检测时，必须保证
   中英文文案键值完整且可回退到中文。

4. **涉及可配置文本**（提示词模板、系统提示、预设文案）时，必须
   按双语结构设计，禁止把单一语言硬编码为唯一值。

## 推荐实现方式

### 集中管理文案

在 `src/i18n/` 目录下统一管理：

```text
src/i18n/
├── index.ts          # 导出 t() 函数和当前语言
├── zh.ts             # 中文文案
└── en.ts             # 英文文案
```

### 文案结构示例

```typescript
// src/i18n/zh.ts
export const zh = {
    settings: {
        title: '插件设置',
        debugMode: '调试模式',
        debugModeDesc: '启用后在控制台输出调试信息',
        debugLevel: '调试级别',
        debugLevelDesc: '控制输出的最低日志级别',
    },
    commands: {
        openPanel: '打开面板',
        refreshData: '刷新数据',
    },
    notices: {
        saveSuccess: '设置已保存',
        saveError: '保存失败：{reason}',
    },
} as const;

// src/i18n/en.ts
export const en: typeof zh = {
    settings: {
        title: 'Plugin Settings',
        debugMode: 'Debug Mode',
        debugModeDesc: 'Output debug info to console when enabled',
        debugLevel: 'Debug Level',
        debugLevelDesc: 'Minimum log level for output',
    },
    commands: {
        openPanel: 'Open Panel',
        refreshData: 'Refresh Data',
    },
    notices: {
        saveSuccess: 'Settings saved',
        saveError: 'Save failed: {reason}',
    },
};
```

### 使用方式

```typescript
// src/i18n/index.ts
import { zh } from './zh';
import { en } from './en';

type Lang = 'zh' | 'en';

const translations: Record<Lang, typeof zh> = { zh, en };
let currentLang: Lang = 'zh'; // 默认中文

export function setLang(lang: Lang): void {
    currentLang = lang;
}

export function t(): typeof zh {
    return translations[currentLang];
}
```

```typescript
// 在业务代码中使用
import { t } from '../i18n';

new Notice(t().notices.saveSuccess);
```

## 命名与扩展预留

- 文案 key 使用英文点分命名（`settings.title`），不使用中文 key
- `typeof zh` 作为类型基准，`en` 必须实现完全相同的结构
- 后续新增语言只需添加新文件并注册到 `translations` 映射中
- 禁止在组件和业务逻辑中散落硬编码文本

## 提交前检查

- [ ] 所有新增用户可见文本已同步添加到 `zh.ts` 和 `en.ts`
- [ ] `en.ts` 的类型检查通过（`typeof zh` 约束，缺少 key 会编译报错）
- [ ] 默认语言仍为中文
- [ ] 无散落在组件中的硬编码中/英文文本

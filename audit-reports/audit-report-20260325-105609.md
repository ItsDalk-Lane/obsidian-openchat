## 修复记录

### 1. 启动链与主插件职责收敛
- **问题**：`main.ts` 同时承担设置控制、启动清理、目录迁移、MCP 初始化等多项业务职责。
- **规则来源**：项目 CLAUDE.md 第 1、2、12 条。
- **修复方式**：新增 `PluginStartupCoordinator` 与 `PluginSettingsController`，把延迟初始化、目录迁移与设置控制从插件入口下沉；`main.ts` 保留注册和生命周期职责。
- **关键文件**：`src/main.ts`、`src/core/PluginStartupCoordinator.ts`、`src/settings/PluginSettingsController.ts`

### 2. 统一日志治理与脚本输出门禁
- **问题**：大量直接 `console` 输出绕过统一门禁，构建脚本缺少受控日志层。
- **规则来源**：项目 CLAUDE.md 第 7 条；全局 CLAUDE.md 异常处理与调试约束。
- **修复方式**：补齐 `DebugLogger` 行为，新增 `scripts/script-logger.mjs`，将构建脚本、provider、chat 服务、组件及工具侧的调试输出收口到统一日志入口。
- **关键文件**：`src/utils/DebugLogger.ts`、`scripts/script-logger.mjs`、`esbuild.config.mjs`、`scripts/build-and-sync.mjs`、`scripts/copy-to-vault.mjs`、`scripts/provider-regression.mjs`

### 3. 双语支持与 i18n 闭环
- **问题**：用户可见文本存在硬编码，设置面板与聊天组件部分文案未统一走 i18n。
- **规则来源**：项目 CLAUDE.md 第 13.1 条。
- **修复方式**：补齐 ai-runtime locale，扩展 `src/i18n` 文案，新增 fragments 分片文件，将 provider 配置、quick actions、chat 组件和 Notice 文案统一迁移到 i18n 入口。
- **关键文件**：`src/i18n/en.ts`、`src/i18n/local.ts`、`src/i18n/locals.ts`、`src/i18n/zh.ts`、`src/i18n/zhTw.ts`、`src/i18n/fragments/**`、`src/i18n/ai-runtime/**`

### 4. Provider 与 Poe 模块拆分
- **问题**：多家 provider 存在能力判断、请求构造和消息转换逻辑耦合；`src/LLMProviders/poe.ts` 体量过大。
- **规则来源**：全局 CLAUDE.md 500 行限制、单一职责原则；项目 CLAUDE.md 模块化要求。
- **修复方式**：拆出 provider 辅助模块；将 Poe provider 拆为 chat runners、responses runners、hybrid runner、message transforms、runner shared 等多个文件，保留 `poe.ts` 为薄入口调度层。
- **关键文件**：`src/LLMProviders/poe.ts`、`src/LLMProviders/poeChatRunners.ts`、`src/LLMProviders/poeHybridRunner.ts`、`src/LLMProviders/poeMcpRunners.ts`、`src/LLMProviders/poeResponsesRunners.ts`、`src/LLMProviders/poeResponsesRequestRunners.ts`、`src/LLMProviders/poeMessageTransforms.ts`、`src/LLMProviders/poeRunnerShared.ts`

### 5. ChatService 与编辑器侧大文件拆分
- **问题**：`ChatService.ts`、编辑器集成与快捷操作面板职责集中，影响可维护性。
- **规则来源**：全局 CLAUDE.md 单文件 500 行限制、模块化和 DRY 原则。
- **修复方式**：把 ChatService 拆为 `ChatServiceCore`、`ChatServiceMid`、`ChatServiceOps`，并抽出 message operations、provider messages、history formatting、summary 等辅助模块；编辑器侧拆出 base integration、modify requester、toolbar hook 和 quick action 子模块。
- **关键文件**：`src/core/chat/services/ChatService.ts`、`src/core/chat/services/ChatServiceCore.ts`、`src/core/chat/services/ChatServiceMid.ts`、`src/core/chat/services/ChatServiceOps.ts`、`src/core/chat/services/chat*.ts`、`src/editor/chat/ChatEditorIntegrationBase.tsx`、`src/editor/chat/ChatEditorModifyRequester.ts`、`src/editor/selectionToolbar/useSelectionToolbar.ts`

### 6. 设置面板、MCP 与 Vault 工具链重构
- **问题**：provider 配置、quick actions、MCP 调用和 vault 工具聚合文件职责混杂。
- **规则来源**：项目 CLAUDE.md 模块化要求；全局 CLAUDE.md 单一职责与可维护性约束。
- **修复方式**：新增 provider-config 子模块、quick-actions 子模块、MCP helper 模块和 vault filesystem/query 子模块，按功能拆分渲染、校验、解析和执行逻辑。
- **关键文件**：`src/components/settings-components/provider-config/**`、`src/components/settings-components/quick-actions/**`、`src/services/mcp/mcpToolArgHelpers.ts`、`src/services/mcp/mcpToolCallHandlerInternals.ts`、`src/services/mcp/mcpTransportFactory.ts`、`src/tools/vault/filesystem*.ts`、`src/tools/vault/vault-query-*.ts`

### 7. 构建与验证结果
- **正式验证**：执行 `npm run build`，构建、归档与同步流程全部通过。
